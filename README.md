## Getting Started


### Prerequisites

- Docker - [Download Docker](https://www.docker.com/get-started)
- Docker Compose - [Download Docker Compose](https://docs.docker.com/compose/install/)


### Installation

1. **Clone the repository**

   ```bash
   git clone git@github.com:mateoscarlosj/Sprout-Exam.git
   cd Sprout-Exam

2. **Build and run the application**
   ```bash
   docker-compose up --build

3. **Access the application**
- Frontend: http://localhost:8080
- Backend: http://localhost:8000

## Future Improvements

If we were to deploy this application to production, the next improvements I would prioritize include:

1. **Enhancing Login Security**: Implement more robust authentication and authorization mechanisms to protect user credentials and sensitive data.

2. **Refining Dashboard Design**: Improve the user interface and user experience of the dashboard to make it more intuitive and visually appealing.

3. **Improving Error Handling**: Develop comprehensive error handling strategies to provide clear and actionable feedback to users and ensure system reliability.
4. **Creating Global Design Components**:  Develop a set of reusable design components that can be used by future developers to maintain consistency and efficiency in the application's design.
